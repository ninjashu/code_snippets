What is WdgM? — Architecture overview

WdgM (Watchdog Manager) sits between SW-Cs/BSW and the hardware watchdog. It does NOT trigger the watchdog directly — it monitors software health, then tells WdgIf to keep triggering (or stop, causing a reset).

Core idea: SW-C calls WdgM_CheckpointReached() → WdgM judges health → WdgM_MainFunction() tells WdgIf_SetTriggerCondition() → HW watchdog lives or resets ECU.

Layer position

↑
SW-Cs / CDDs — call CheckpointReached via RTE ports
M
WdgM — evaluates Alive/Deadline/Logical supervision
I
WdgIf — abstracts multiple watchdog drivers
D
Wdg Driver — HW watchdog trigger

Key dependencies

  • EcuM — initializes Wdg driver BEFORE WdgM_Init
  • OS — provides GetElapsedValue() for deadline timestamps, task scheduling
  • DEM — receives WDGM_E_SUPERVISION after reset-due-to-WdgM
  • DET — receives dev/runtime error codes
  • RTE — routes CheckpointReached calls, mode notifications
  • MCU — optional immediate reset via Mcu_PerformReset()

Key constraints (exam critical)

  • WdgM_MainFunction task must NOT contain supervised entities
  • WdgM_Init/DeInit/SetMode must NOT be called concurrently with MainFunction
  • Supervision starts at FIRST call to MainFunction after Init
  • WdgM can't use RTE APIs before first MainFunction call
  • Mode switch only allowed in GLOBAL_OK or GLOBAL_FAILED

Three supervision types

  • Alive — "Is it running often enough?" Counts checkpoint calls per reference cycle window
  • Deadline — "Is the timing between events correct?" Checks min/max time between Start→End checkpoints
  • Logical — "Is execution order correct?" Validates checkpoint sequence against a graph

All three results combine per Supervised Entity → Local Status → Global Status

Supervised Entity (SE) structure

Each SE has one or more Checkpoints. Checkpoints connect via Transitions. SE gets a unique WdgMSupervisedEntityId (uint16). Checkpoint identified by pair of SEID + CheckpointID.

Internal Graph: transitions within one SE, mode-independent. External Graph: transitions across SEs, mode-dependent. External graphs enable cross-SE sequence validation.

Supervision functions

Alive supervision

Checks cyclic execution reliability. Did this checkpoint get called the right number of times per supervision cycle?

Configuration params (per checkpoint, per mode)

WdgMExpectedAliveIndications (EAI)
Expected count of WdgM_CheckpointReached calls per WdgMSupervisionReferenceCycle
WdgMMinMargin
Acceptable negative variation: min acceptable = EAI - MinMargin
WdgMMaxMargin
Acceptable positive variation: max acceptable = EAI + MaxMargin
WdgMSupervisionReferenceCycle
Multiple of MainFunction period at which counter is evaluated
WdgMFailedAliveSupervisionRefCycleTol
How many consecutive FAILED reference cycles tolerated before EXPIRED. 0 = immediate EXPIRED

Algorithm

1
SW-C calls WdgM_CheckpointReached(SEID, CPID) → alive counter for that checkpoint increments
2
At each WdgMSupervisionReferenceCycle, MainFunction evaluates counter
3
If counter within [EAI-Min, EAI+Max] → result=correct; else → result=incorrect
4
Incorrect result triggers FAILED counter increment (or EXPIRED if tolerance=0)
Only Alive has tolerance (FAILED state buffer). Deadline and Logical failures go straight to EXPIRED with no FAILED stop.

Deadline supervision

Checks non-cyclic timing: time between a Start checkpoint and End checkpoint must be within [Min, Max].

Configuration params (per transition, per mode)

WdgMDeadlineStartRef
Reference to Start Checkpoint
WdgMDeadlineStopRef
Reference to End Checkpoint
WdgMDeadlineMin
Minimum allowed time between Start and End (seconds). Early arrival = fail
WdgMDeadlineMax
Maximum allowed time. 'INF' disables upper check. Late/missing arrival = fail
WdgMOSCounter
OS counter used for timestamp (GetElapsedValue). Required when any Deadline configured
WdgMEnableTimeoutDetection
If true: MainFunction also checks if End never arrives (timeout detection)

Algorithm (SWS_WdgM_00228–00299)

1
Start checkpoint reached → timestamp saved (OS ticks)
2
Start reached again before End → timestamp overwritten (NOT an error)
3
End checkpoint reached + timestamp ≠ 0 → time diff measured, timestamp cleared
4
End reached with timestamp = 0 → ignored silently (no error)
5
Time diff outside [Min, Max] → result=incorrect (→ immediate EXPIRED)
6
Timeout detection: MainFunction checks if timestamp ≠ 0 and elapsed > Max → incorrect

Deadline uses OS ticks scaled by OsSecondsPerTick. OS counter must be accessible from both the SE's OsApplication AND the WdgM_MainFunction's OsApplication when timeout detection is enabled.

Logical supervision

Verifies execution sequence correctness. Checkpoints form a graph; WdgM validates each reported checkpoint is a valid successor of the previous one.

Key concepts

Internal Graph
Transitions within one SE. Mode-independent (always same graph, activation toggled by mode)
External Graph
Transitions across multiple SEs. Mode-dependent (configured per WdgMMode)
Initial Checkpoint
First valid checkpoint in a graph. Activates the Activity Flag for that graph
Final Checkpoint
Resets Activity Flag to false (graph ready to restart)
Activity Flag
One per graph. False = waiting for initial checkpoint. True = actively checking sequence

Algorithm

1
Activity Flag = false → only Initial Checkpoint is valid. Non-initial = incorrect
2
Initial Checkpoint reached → Activity Flag = true, stored as current checkpoint
3
Next checkpoint reached → verify transition (stored→new) exists in graph. Valid → correct, store new. Invalid → incorrect (and stays incorrect)
4
Final Checkpoint valid → Activity Flag = false
5
Checkpoint not in any graph → ignored by Logical (may still affect Alive/Deadline)
Incorrect External Transition: the SE that reported the wrong checkpoint is marked as erroneous — not the SE that sent the previous checkpoint.

API specification

Service ID: 0x00 | Sync | Non-Reentrant
void WdgM_Init(const WdgM_ConfigType* ConfigPtr)
Initializes WdgM. Sets all Local statuses: active SEs → OK, inactive → DEACTIVATED. Global → OK. Failed ref cycle counters → 0. Clears non-init RAM (first expired SEID).
Precondition: WdgIf and OS must be initialized first (EcuM responsibility). Supervision starts at FIRST MainFunction call, not here. If WdgIf_SetMode fails during Init → GLOBAL_STOPPED immediately.
Service ID: 0x01 | Sync | Non-Reentrant
void WdgM_DeInit(void)
De-initializes WdgM. Performs a mode switch to update watchdog trigger conditions (for windowed watchdog compatibility). Global status → DEACTIVATED. Stops all supervision.
Cannot be called when in EXPIRED or STOPPED — risks blocking error recovery. EcuM calls this during SLEEP entry.
Service ID: 0x02 | Sync | Reentrant
void WdgM_GetVersionInfo(Std_VersionInfoType* VersionInfo)
Returns module version info. NULL pointer → DET error WDGM_E_INV_POINTER.
Service ID: 0x03 | Sync | Non-Reentrant | Returns Std_ReturnType
Std_ReturnType WdgM_SetMode(WdgM_ModeType Mode)
Switches WdgM mode. Updates supervision parameters and watchdog trigger conditions. Returns E_OK on success, E_NOT_OK on failure.
Only executes if Global = OK or FAILED. In other states → returns without action (SWS_WdgM_00316). WdgIf_SetMode failure → Global_STOPPED (runtime error WDGM_E_SET_MODE). Mode disabling watchdog when WdgMOffModeEnabled=FALSE → E_NOT_OK + WDGM_E_DISABLE_NOT_ALLOWED.
Service ID: 0x0b | Sync | Reentrant | Returns Std_ReturnType
Std_ReturnType WdgM_GetMode(WdgM_ModeType* Mode)
Returns current active mode. If SetMode is in progress, returns previous mode until new mode fully activated.
Service ID: 0x0e | Sync | Reentrant | Returns Std_ReturnType
Std_ReturnType WdgM_CheckpointReached(WdgM_SupervisedEntityIdType SEID, WdgM_CheckpointIdType CheckpointID)
Called by SW-C when code reaches a checkpoint. Does three things: (1) increment alive counter, (2) execute Deadline supervision (early arrival + delay detection), (3) execute Logical supervision.
Reentrant — safe to call from multiple SEs concurrently. Returns E_NOT_OK if SE is deactivated in current mode (WDGM_E_SEDEACTIVATED). Timeout detection (missing End) is handled in MainFunction, not here.
Service ID: 0x0c | Sync | Reentrant | Returns Std_ReturnType
Std_ReturnType WdgM_GetLocalStatus(WdgM_SupervisedEntityIdType SEID, WdgM_LocalStatusType* Status)
Returns current local supervision status for a specific SE. Used by SW-Cs for self-diagnostics or by integrators for testing.
Service ID: 0x0d | Sync | Reentrant | Returns Std_ReturnType
Std_ReturnType WdgM_GetGlobalStatus(WdgM_GlobalStatusType* Status)
Returns current global supervision status. Key API for integration testing — confirms WdgM health at system level.
Service ID: 0x0f | Sync | Non-Reentrant | Returns void
void WdgM_PerformReset(void)
Forces immediate watchdog reset. Sets all watchdog trigger conditions to 0. After this call, WdgM stops all further trigger updates regardless of Global Status.
Use for deliberate forced reset. Not the same as letting GLOBAL_STOPPED happen — this is an explicit API call. No RTE mode notification is sent.
Service ID: 0x10 | Sync | Reentrant | Returns Std_ReturnType
Std_ReturnType WdgM_GetFirstExpiredSEID(WdgM_SupervisedEntityIdType* SEID)
Returns the SEID that first went EXPIRED in the current operating cycle. Read from non-initialized RAM (persists across reset). Stored as double-inverted value for data integrity check.
Available BEFORE WdgM_Init — critical for post-reset diagnosis. If value and ~value don't match → E_NOT_OK, SEID=0. Call this in startup to determine why last reset occurred.
Service ID: 0x08 | Called by BSW Scheduler
void WdgM_MainFunction(void)
Core periodic function. Executes: (1) Alive supervision evaluation, (2) Deadline timeout detection, (3) Local Status state machine update for all active SEs, (4) Global Status update, (5) WdgIf_SetTriggerCondition call.
Must be in a task that contains NO supervised entities. Supervision starts at first call after Init. Multiple partitions → WdgM_MainFunction_<shortName> naming.

Sequence diagrams

Normal operation sequence

1
EcuM startup: Wdg driver init → WdgIf init → WdgM_Init(ConfigPtr) called. Global=OK, all active SEs=LOCAL_OK.
2
BSW Scheduler: WdgM_MainFunction() called periodically (first call activates supervision)
3
SW-C Runnable: WdgM_CheckpointReached(SEID, CPID) at each code checkpoint → counters updated, Deadline/Logical evaluated
4
MainFunction cycle: Evaluate Alive counters at reference cycle → compute Local Status → compute Global Status → call WdgIf_SetTriggerCondition(deviceIdx, timeout)
5
Mode switch (e.g. BswM): WdgM_SetMode(newMode) → updates SE activation, watchdog params
6
EcuM sleep: WdgM_DeInit() → performs internal mode switch for windowed WDG, deactivates WdgM

Failure detection and reset sequence

1
SE fails supervision → CheckpointReached returns Deadline/Logical incorrect result
2
MainFunction: Local status → EXPIRED (for Deadline/Logical); or FAILED→EXPIRED for Alive
3
Global status update: one SE EXPIRED + WdgMExpiredSupervisionCycleTol=0 → GLOBAL_STOPPED
4
WdgIf_SetTriggerCondition(deviceIdx, 0) → HW watchdog stop being fed
5
If WdgMImmediateReset=TRUE: Mcu_PerformReset() called immediately
6
RTE mode mechanism: SW-Cs notified of SUPERVISION_STOPPED via mode port
7
HW watchdog expires → ECU reset
8
After reset: Before WdgM_Init, call WdgM_GetFirstExpiredSEID() to identify culprit SE
9
First MainFunction call after Init: checks reset cause → reports WDGM_E_SUPERVISION FAILED to DEM, then reports PASSED

Alive supervision internal sequence (per MainFunction call)

1
Check if current cycle is a supervision reference cycle for this checkpoint
2
If yes: read alive counter, compare to [EAI-Min, EAI+Max]
3
Counter in range → result=correct, reset counter for next window
4
Counter out of range → result=incorrect
5
If tolerance>0 and result=incorrect: FAILED state + increment failed cycle counter
6
If counter reaches tolerance limit → EXPIRED
7
If SE was FAILED and result=correct: decrement counter; if counter=0 → OK

Logical supervision sequence (per CheckpointReached)

1
Determine which Graph(s) the reported CPID belongs to
2
For each Graph: check Activity Flag
3
Flag=false: only Initial Checkpoint valid → if initial, set Flag=true, store CP. If non-initial → incorrect
4
Flag=true: check if transition (stored→reported) exists in graph
5
Valid transition → correct, update stored CP
6
Invalid transition → incorrect, result stays incorrect for remainder
7
If reported CP is Final: set Flag=false (graph ready to restart)

Production error (DEM)

WDGM_E_SUPERVISION
—
Global reached STOPPED → watchdog reset occurred. Reported FAILED in first MainFunction after restart. Then reported PASSED immediately after.
DTC from WDGM_E_SUPERVISION never shows "confirmed" — it resets every startup. Post-reset diagnosis: use WdgM_GetFirstExpiredSEID() + DEM event status.

Debugging checklist

  • WdgM reset happening? → Check WdgM_GetFirstExpiredSEID() before Init, then DEM for WDGM_E_SUPERVISION
  • CheckpointReached returning E_NOT_OK? → SE may be deactivated in current mode; check WdgM_GetMode() and config
  • Alive supervision failing spuriously? → Check task jitter vs WdgMSupervisionReferenceCycle; check Min/MaxMargin tolerances
  • Deadline supervision triggering? → Check OS counter resolution (OsSecondsPerTick); check timestamp = 0 edge cases
  • Logical supervision incorrect? → Trace CheckpointReached call sequence vs configured graph; check Initial Checkpoint is called first
  • Mode switch failing? → Verify Global is OK or FAILED; check WdgMOffModeEnabled; check WdgIf_SetMode return
  • MainFunction task deadlock? → WdgM_MainFunction task must not contain any supervised runnables

Configuration hierarchy

Module-level switches: WdgMDevErrorDetect, WdgMOffModeEnabled, WdgMImmediateReset, WdgMVersionInfoApi, WdgMDemEventParameterRefs (link to DEM event for WDGM_E_SUPERVISION)

WdgMGeneral → WdgMSupervisedEntity [1..N]

One container per SE. Parameters: WdgMSupervisedEntityId (unique uint16), and nested WdgMCheckpoint containers.

Top-level configuration set. Contains WdgMInitialMode reference + all WdgMMode containers.

Step-by-step DaVinci setup

1
Add WdgM BSW module to your ECU configuration. DaVinci auto-creates WdgMGeneral and WdgMConfigSet.
2
Enable WdgMDevErrorDetect during development. Set WdgMOffModeEnabled=FALSE for safety-relevant systems.
3
Create Supervised Entities: Right-click WdgMSupervisedEntity → Add. Assign unique WdgMSupervisedEntityId. Add checkpoints under each SE.
4
Configure Internal Transitions for Logical supervision of internal graphs. Mark initial/final checkpoints.
5
Create Modes under WdgMConfigSet. Set WdgMInitialMode reference. Modes define: which SEs active, supervision params, watchdog modes.
6
Configure WdgMLocalStatusParams per active SE per mode. Link SE reference. Set alive tolerance, expired tolerance.
7
Configure supervision functions per mode: Add WdgMAliveSupervision (link checkpoint, set EAI/margins/refcycle), WdgMDeadlineSupervision (link Start/End CPs, set timing, link OS counter), WdgMExternalLogicalSupervision (define external graph transitions).
8
Configure WdgMTrigger: link watchdog device, set mode (FAST/SLOW/OFF) and trigger condition value per mode.
9
Link DEM event: In WdgMGeneral → WdgMDemEventParameterRefs, reference the DEM event for WDGM_E_SUPERVISION.
10
Configure OS counters (in OS configurator): Add OsCounter, set OsSecondsPerTick. Reference in WdgMDeadlineSupervision → WdgMOSCounter. Ensure OsCounterAccessingApplication includes both SE's partition and MainFunction's partition.
11
Generate code. DaVinci generates WdgM_Cfg.h/c with all IDs and parameters. SW-C port connections auto-generate SEID/CPID constants.
12
OS task configuration: Map WdgM_MainFunction to a separate task from all supervised entities. Set period to match supervision cycle requirements.

Integration testing checklist

  • Verify WdgM_GetGlobalStatus() = WDGM_GLOBAL_STATUS_OK after full system startup
  • Test: disable one SE's checkpoint call → verify local EXPIRED → global STOPPED → ECU reset within expected time
  • Test: call checkpoint too fast/slow → verify Alive supervision response
  • Test: call checkpoint out of order → verify Logical supervision EXPIRED
  • Test: WdgM_GetFirstExpiredSEID() post-reset → correct SEID returned
  • Test: DEM event WDGM_E_SUPERVISION FAILED present after WDG reset, PASSED after normal start
  • Test: mode switch changes supervision params and watchdog trigger condition
  • Verify WdgM_MainFunction task never contains supervised runnables
  • Verify OS counter accessible from both partitions (if deadline supervision across partitions)
