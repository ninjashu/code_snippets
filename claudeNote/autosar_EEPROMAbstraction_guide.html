What is EA and where does it sit?

EA (EEPROM Abstraction) is a BSW module in AUTOSAR Classic Platform. It hides device-specific EEPROM addressing from upper layers and provides a uniform virtual 32-bit address space.

Key concepts to memorize

Virtual address = 32 bits

16-bit block number + 16-bit offset inside block. Block number encodes dataset/redundant copy bits (NVM_DATASET_SELECTION_BITS).

block number: 16 bitoffset: 16 bit

Physical address derived by EA

EA combines block number + offset → calls MemAcc which maps to real EEPROM hardware address. Exact mapping is implementation-specific.

Virtual page size

All blocks aligned to EaVirtualPageSize. Must be integer multiple of physical page size. Block sizes must be integer multiples of virtual page size.

If virtual page = 8 bytes, valid block sizes: 8,16,24... NOT 10,20.

One job at a time

EA has NO job queue. NvM must serialize jobs. EA does not check if busy before accepting — that is NvM's job. But APIs do raise EA_E_BUSY runtime error if called while busy.

Asynchronous execution

All data ops (Read, Write, Cancel, InvalidateBlock, EraseImmediateBlock) are ASYNC. Ea_Init and GetStatus/GetJobResult/GetVersionInfo are SYNC.

Virtually unlimited erase cycles

EA spreads writes across multiple physical areas when device endurance < required write cycles (EaNumberOfWriteCycles vs EepAllowedWriteCycles).

Module dependencies

  • MemAcc — mandatory lower layer since R21-11 (replaced direct Eep calls)
  • NvM — upper layer caller; serializes all jobs
  • MemIf — routing interface; NvM calls MemIf, MemIf calls EA
  • Det — development error reporting (optional, enabled by EaDevErrorDetect)
  • SchM_Ea.h — scheduler integration for Ea_MainFunction

Addressing scheme

Virtual address structure

[ 16-bit block number | 16-bit offset ]

Block number encodes: actual block ID + dataset index bits (NVM_DATASET_SELECTION_BITS from MSB side). Only non-dataset bits used for address calc.

Example: 4 dataset bits → 12 bits for block, 4 bits for dataset. Max 4094 NvM blocks, 16 datasets each.

Block number rules

  • 0x0000 and 0xFFFF FORBIDDEN — never configure
  • Min valid: 2^NVM_DATASET_SELECTION_BITS
  • Max valid: 0xFFFF − 2^NVM_DATASET_SELECTION_BITS
  • Blocks must NOT overlap
  • Blocks must NOT contain one another

Block size alignment rule

Every block size must be integer multiple of EaVirtualPageSize. Virtual page size must be integer multiple of physical page size. If block 100 bytes and page 8 bytes → 100 is NOT valid (not multiple of 8). Must round up to 104.

Block #1 size=32, page=8 → uses 4 virtual pages (block numbers 1,2,3,4 "blocked"). Block #5 size=100 → 13 pages (block numbers 5..17 blocked). Next available = block #18.

Block consistency management

  • When write STARTS → EA marks block inconsistent (stored in NV memory, reset-resistant)
  • When write SUCCEEDS → EA marks block consistent
  • If power loss during write → block stays inconsistent → read returns MEMIF_BLOCK_INCONSISTENT
  • Inconsistent ≠ Invalidated: EA distinguishes these two states
  • Invalidated block → EA explicitly marks it invalid via Ea_InvalidateBlock
  • Reading invalidated → MEMIF_BLOCK_INVALID
  • Reading inconsistent (or block not found) → MEMIF_BLOCK_INCONSISTENT
Upper layer MUST NOT use data buffer contents when job result is BLOCK_INCONSISTENT or BLOCK_INVALID.

Immediate data handling

Immediate data blocks (EaImmediateData = TRUE) must be writable WITHOUT prior erase — pre-erased area reserved. Priority: NvM cancels any ongoing lower-priority job before writing immediate data.

  • Ea_EraseImmediateBlock() — pre-erases area, called by diagnostics/system services
  • Ea_Cancel() called by NvM synchronously before immediate write
  • Hardware may still be completing last physical op (page write) — EA must wait for that
  • Immediate write cannot be delayed by internal mgmt ops

Write cycle spreading (endurance management)

If EepAllowedWriteCycles (EEPROM hw endurance) < EaNumberOfWriteCycles (required by config), EA must internally provide multiple physical areas and rotate access — wear leveling.

Example: block needs 500k cycles, EEPROM rated 100k → EA uses at least 5 physical locations, round-robin.

Buffer alignment (added R21-11)

  • EaBufferAlignmentValue — internal buffers aligned to this (inherited from MemAccBufferAlignmentValue)
  • EaMinimumReadPageSize — read requests aligned to this page size

Internal management vs upper-layer job result

EA may run internal ops (garbage collection, wear leveling) during NvM-requested jobs. These internal ops must NOT affect the job result returned by Ea_GetJobResult(). EA needs a second set of internal variables for this purpose.

Exam question: "Does garbage collection affect Ea_GetJobResult()?" — Answer: NO. Only upper-layer jobs affect job result.

Ea_Init sequence

1EcuM calls Ea_Init(NULL_PTR)
2EA sets module state: UNINIT → BUSY_INTERNAL
3aIf init completes synchronously: state → IDLE (done)
3bIf init needs async work: remains BUSY_INTERNAL until Ea_MainFunction() completes it → IDLE
EA_Init NOT called by NvM. EcuM calls it during startup. NvM may not start until EA is IDLE.

Ea_Write sequence (complete flow)

1NvM calls MemIf_Write() → MemIf routes to Ea_Write(BlockNumber, DataBufferPtr)
2EA checks state (IDLE or BUSY_INTERNAL), checks block number, checks DataBufferPtr ≠ NULL
3EA calculates physical address (block offset fixed to 0), sets length = EaBlockSize
4EA copies params to internal vars, marks block INCONSISTENT, sets state = BUSY, job result = JOB_PENDING, returns E_OK to NvM
5NvM returns; BSW scheduler calls Ea_MainFunction() periodically
6Ea_MainFunction() calls MemAcc_Write() → MemAcc → Mem driver → physical EEPROM
7MemAcc runs its MainFunction loop; Mem performs actual write
8On hw complete: Ea_JobEndNotification() called by MemAcc
9EA: marks block CONSISTENT, sets job result = JOB_OK, state = IDLE
10EA calls NvM_JobEndNotification() → NvM updates its records
Actual data movement only in steps 6-8. Steps 1-4 just register the job. Steps 9-10 are the completion path.

Ea_Cancel sequence

1NvM calls MemIf_Cancel() → MemIf calls Ea_Cancel()
2EA checks: if UNINIT → DET error, return. If NOT BUSY → EA_E_INVALID_CANCEL, return unchanged
3If BUSY: EA calls MemAcc_Cancel() to tell lower layer stop sending new commands
4EA resets internal vars, sets job result = JOB_CANCELED, state = IDLE
5Physical hardware may still complete current page write — EA does NOT abort ongoing HW op
6Ea_JobEndNotification may still arrive from MemAcc — EA handles this gracefully (job result already CANCELED, not PENDING, so notification left untouched)
7NvM can now issue immediate data write (reason for cancel)
Cancel is synchronous in terms of EA state reset but asynchronous w.r.t. hardware. Don't confuse these.

Ea_Read + consistency check sequence

1NvM → MemIf_Read() → Ea_Read(block, offset, buffer, length)
2EA validates params (DET checks), calculates address, sets BUSY + JOB_PENDING, returns E_OK
3Ea_MainFunction: check if block was invalidated → if yes, set BLOCK_INVALID, call error notification
4Ea_MainFunction: call MemAcc_Read() → actual data transferred to buffer
5After read: EA checks block consistency markers in management data
6aConsistent → set JOB_OK, call NvM_JobEndNotification
6bInconsistent or not found → set BLOCK_INCONSISTENT, call NvM_JobErrorNotification
Buffer contents are transferred even if inconsistent — but NvM MUST ignore buffer contents on error result.

Vector DaVinci Configurator — EA module step by step

Step 1: Add EA module to BSW configuration

In DaVinci Developer or Configurator Pro: BSW Module Selection → add Ea. This creates the Ea container in ECUC dataset.

Step 2: Configure EaGeneral container

EaGeneral parameters
EaDevErrorDetect bool TRUE during development/integration testing. FALSE in production. Enables DET calls and parameter validation.
EaVirtualPageSize uint Must match or be multiple of physical EEPROM page size. Check EEPROM datasheet. Typical values: 1, 2, 4, 8 bytes. All block sizes must be multiples of this.
EaMainFunctionPeriod float In seconds. Match to EEPROM driver cycle. Typical: 0.005 (5ms). Set same period in OS task that calls Ea_MainFunction.
EaPollingMode bool TRUE = polling (no callbacks from Eep/MemAcc). FALSE = callback mode (Ea_JobEndNotification used). Match to MemAcc/Eep config.
EaNvmJobEndNotification fn name Set to NvM_JobEndNotification. This is the callback EA calls when job succeeds. Must match NvM config.
EaNvmJobErrorNotification fn name Set to NvM_JobErrorNotification. Called on job failure. Must match NvM config.
EaVersionInfoApi bool TRUE to enable Ea_GetVersionInfo(). Usually FALSE in production.
EaBufferAlignmentValue ref Reference to MemAccAddressAreaConfiguration → inherits MemAccBufferAlignmentValue. Set in R21-11+.
EaMinimumReadPageSize uint Read requests aligned to this. Match to EEPROM minimum addressable read unit. Draft status in R24-11.

Step 3: Configure EaBlockConfiguration (one per logical block)

EaBlockConfiguration parameters
EaBlockNumber uint 16-bit block ID. Range: [2^NVM_DATASET_SELECTION_BITS .. 0xFFFF−2^NVM_DATASET_SELECTION_BITS]. Must NOT be 0x0000 or 0xFFFF. Must match NvM block config. DaVinci generates symbolic name.
EaBlockSize uint Size in bytes. MUST be integer multiple of EaVirtualPageSize. DaVinci will warn if not aligned. Match to NvM configured block size.
EaImmediateData bool TRUE only for blocks that must be writable without erase (immediate/safety data). Pre-erased area reserved. Required for Ea_EraseImmediateBlock to work.
EaNumberOfWriteCycles uint32 Expected write cycles for this block. If > EepAllowedWriteCycles → EA must implement wear leveling. Set realistically — overestimating wastes flash area.
EaMemAccAddressArea ref Reference to MemAccAddressAreaConfiguration. Used R21-11+. Replaces EaDeviceIndex (obsolete). Tells EA which MemAcc address area this block lives in.
EaDeviceIndex obsolete Old reference to EepGeneral. Now obsolete — use EaMemAccAddressArea instead. DaVinci may show this as deprecated.

Step 4: Published Information (EaPublishedInformation — read-only)

Set by EA vendor/implementer, not by user:

  • EaBlockOverhead — management bytes per block (vendor specifies)
  • EaPageOverhead — management bytes per page (vendor specifies)

These values needed by NvM to calculate total EEPROM memory layout. Import from vendor delivery.

Step 5: OS task / scheduling setup

  • Create BSW task or use existing 5ms/10ms task
  • Add Ea_MainFunction() call in that task
  • Period must match EaMainFunctionPeriod value
  • In DaVinci: SchM_Ea.h included automatically; configure in OS/RTE settings

Step 6: MemIf / NvM cross-references

  • In NvM config: NvMBlockDescriptor → NvMDeviceId must reference MemIf slot pointing to EA
  • In MemIf: device 0 (or N) mapped to EA module
  • Block numbers must be consistent: EaBlockNumber = NvM block number × 2^NVM_DATASET_SELECTION_BITS
  • EaBlockSize must equal NvMNvBlockLength (× dataset count for datasets)

Step 7: Generate code

DaVinci generates: Ea_Cfg.h, Ea_Cfg.c with block table, address calculations, config structs. Do NOT hand-edit generated files.

Common DaVinci mistakes: EaBlockSize not multiple of EaVirtualPageSize (config error), EaBlockNumber overlapping with another block, callback function name mismatch between EA and NvM configs, EaMainFunctionPeriod not matching OS task period.

Integration testing checklist

Startup / init verification

  • Poll Ea_GetStatus() after EcuM init — must reach MEMIF_IDLE before NvM starts
  • If stuck at BUSY_INTERNAL: Ea_MainFunction not called / wrong period
  • MemAcc not initialized before EA? → EA write will fail

Write/Read round-trip test

  • Write pattern to block → read back → compare
  • Check job result after each operation via Ea_GetJobResult()
  • Verify NvM_JobEndNotification fires after each op
  • Verify status returns IDLE after each completed job

Consistency check testing

  • Force power-off mid-write (HW bench or simulation)
  • On next startup: read same block
  • Expected: MEMIF_BLOCK_INCONSISTENT
  • NvM must use default values — trace NvM_JobErrorNotification call

DET testing

  • Call Ea_Read before Ea_Init → expect EA_E_UNINIT
  • Call with invalid block number → EA_E_INVALID_BLOCK_NO
  • Call with NULL buffer → EA_E_PARAM_POINTER
  • Call Ea_Write while busy → EA_E_BUSY (runtime error)
  • Verify Det_ReportError / Det_ReportRuntimeError called with correct args

Cancel testing

  • Issue Ea_Write, immediately call Ea_Cancel before MainFunction runs
  • Job result must become JOB_CANCELED
  • Status must return to IDLE
  • Call Ea_Cancel with no pending job → EA_E_INVALID_CANCEL

Immediate data test

  • Long write in progress (Ea_Write 200-byte block)
  • Trigger immediate data write condition
  • Verify NvM calls Ea_Cancel first
  • Verify immediate block write starts without delay after cancel

Key exam traps

  • Ea_Init called by EcuM, NOT NvM
  • Write offset always 0; Read offset can be anything
  • BUSY_INTERNAL accepts new jobs; BUSY does not
  • Block 0x0000 and 0xFFFF never valid
  • Internal ops (garbage collection) do NOT affect Ea_GetJobResult()
  • Ea_Cancel is async w.r.t. HW but sync w.r.t. EA state reset
  • EaDeviceIndex is OBSOLETE — use EaMemAccAddressArea
  • Ea_SetMode() was removed in R21-11
  • Ea_Cancel() became asynchronous in R21-11
  • No production errors in EA — none, zero
  • Config variant: VARIANT-PRE-COMPILE only (no post-build variant support)
  • Ea_JobEndNotification must be callable on interrupt level
  • Block size must be integer multiple of EaVirtualPageSize — not just ≤
