What is PduR?

PDU Router = central middleman in AUTOSAR communication stack. Sits between upper layers (Com, Dcm) and lower layers (CanIf, FrIf, CanTp). Routes I-PDUs based on static routing table — no dynamic/payload-based routing.

Key facts (exam-ready)

Mandatory PduR present in every AUTOSAR ECU. Cannot skip it.
ID-based Routing uses static I-PDU ID lookup. No payload inspection.
No mix Cannot gateway IF ↔ TP. Only IF↔IF or TP↔TP.
Generic CDD (Complex Device Drivers) can be upper or lower modules.

Module limits (what PduR does NOT do)

PduR does NOT: modify I-PDU payload / extract signals / do checksums / rate-convert / route based on payload content / gateway between IF and TP modules.

State Machine

Only 2 states: PDUR_UNINIT and PDUR_ONLINE. Only one instance of this state machine exists.

Init sequence (SWS_PduR_00334)

BswM calls PduR_Init(ConfigPtr)
↓
Release all buffers (SWS_PduR_00645)
↓
Load default values into Tx buffers (SWS_PduR_00308)
↓
State → PDUR_ONLINE
↓
Init enabled RoutingPathGroups (PduRIsEnabledAtInit=TRUE)
Debug tip: If PduR behaves erratically at startup → check if PduR_Init was called before any Transmit/Receive API. Check DET for PDUR_E_UNINIT (0x01).

Unknown length PDUs (TpSduLength = 0)

Streaming data with unknown length → PduR passes TpSduLength=0 to upper layer. Upper must handle. For Tx: PduR calls LoTp_Transmit with SduLength=0 (SWS_PduR_00822).

Debug TP: BUFREQ_E_BUSY → data not available yet. BUFREQ_E_NOT_OK → retry not possible (rollback failed). BUFREQ_OK with AvailData=0 → buffer empty but connected. If StartOfReception fails → PduR stops forwarding that connection.

I-PDU Gatewaying

Gateway = PduR is receiver AND transmitter simultaneously. Receives from source bus, forwards to destination bus(es). May also forward to upper layers at same time.

TP Gateway (on-the-fly vs direct)

On-the-fly

Start Tx on dest TP before complete I-PDU received. Threshold = PduRTpThreshold bytes. Saves memory. Only 1 dest TP allowed (SWS_PduR_00708). Buffer position tracked per-destination.

Direct GW

Receive full I-PDU first, then call DstLoTp_Transmit. SWS_PduR_00551: called in context of SrcLoTp_RxIndication (E_OK). Supports 1:n.

Buffering Concept

When is buffering mandatory?
Mandatory IF gateway destinations using TRIGGERTRANSMIT data provision
Mandatory All TP gateways
Optional IF gateway with DIRECT provision (no buffer possible)

Buffer types

Global buffer

PduRBuffer not referenced by any PduRDestBufferRef. Can be occupied by ANY routing path dynamically. First-come first-served allocation.

Dedicated buffer

PduRBuffer referenced via PduRDestBufferRef in a routing path. Reserved for that path only. Used for OBD/diagnostic requests (high priority, must not be delayed).

Buffer allocation priority (SWS_PduR_00797, 00798)

I-PDU needs buffering
↓
Check dedicated buffers (PduRDestBufferRef)
↓
If dedicated fits (size ≤ PduRPduMaxLength) → use dedicated
↓ else
Dynamically allocate global buffer (fits required size)

PduRQueueDepth rules

ValueBuffer typeBehavior
0No bufferImmediate forward only (DIRECT provision)
1Last-is-bestSingle slot. Overwrite on new. For TRIGGERTRANSMIT — buffer held until TriggerTransmit call.
>1FIFOQueue depth = max simultaneous buffered PDUs. New appended to tail. FIFO needs concurrency protection!

Fan-in and Multicast queuing strategy

PDUR_COMMON_QUEUE

All sources (fan-in) or destinations (multicast) share ONE FIFO. Required for fan-in. In multicast: one item removed only after ALL destinations fetched it.

PDUR_DEDICATED_QUEUE

Each source/dest has own queue. For multicast only. Each dest independent — one slow dest doesn't block others.

Fan-in constraint: ALL PduRRoutingPaths in a fan-in MUST use PDUR_COMMON_QUEUE. Mixed queuing strategies = config error. SWS_PduR_CONSTR_00871/00872.

Length handling in buffers (SWS_PduR_00746)

When buffered: copy min(received_length, configured_dest_length). Overlength PDU is truncated to dest I-PDU size. Rest dropped. Prevents buffer overrun.

Routing Path Groups

Purpose: Group routing paths that can be enabled/disabled together at runtime. Managed by BswM module. Used for partial networking, network activation/deactivation.

Enable/Disable rules

ConditionRouting path enabled?
Path belongs to NO group✓ Always enabled
Path belongs to ≥1 group and at least one is enabled✓ Enabled (SWS_PduR_00891)
Path belongs to groups and ALL are disabled✗ Disabled (SWS_PduR_00892)

Initialization

PduRIsEnabledAtInit = TRUE

Group enabled after PduR_Init. SWS_PduR_00894.

PduRIsEnabledAtInit = FALSE

Group disabled after PduR_Init. SWS_PduR_00895.

↓

Test scenario: Disable a group with active TP Rx ongoing → RxIndication still comes through but StartOfReception/CopyRxData for new sessions blocked. Test this edge case!

Production Error (SWS_PduR_00921)

PDUR_E_ROUTING_PATH_BUFFER_OVERFLOW

Reported to DEM (not DET). FAIL: every PduRTxBuffer becomes occupied. PASS: at least one PduRTxBuffer free. Triggered on each PduR API call. Use for field-level buffer exhaustion monitoring.

Error instance ID (SWS_PduR_00824)

PduR uses the caller module's moduleId as instanceId when calling Det_ReportError. E.g., error in PduR_FrIfRxIndication → instanceId = FrIf module ID (61). This helps pinpoint WHICH interface triggered the error.

DET enable/disable

PduRDevErrorDetect (config param) — when FALSE, dev errors silently ignored. Always enable in development/test builds. Production may disable for performance.
